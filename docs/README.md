# S3DOT 開発記録

S3DOTの開発・運用で得られた情報を、後から検索・引き継ぎできる形で保存します。

## 分類

| フォルダ | 内容 | 命名例 |
| --- | --- | --- |
| `daily/` | 日々の作業記録 | `2026-07-16.md` |
| `weekly/` | 1週間の成果と次週計画 | `2026-W29.md` |
| `monthly/` | 月次の成果・課題・翌月目標 | `2026-07.md` |
| `meeting/` | 会議・AI相談の決定事項 | `2026-07-16_mcp導入方針.md` |
| `ideas/` | 未着手のアイデア・仮説 | `建設AI事務員.md` |
| `bugs/` | 不具合の症状・原因・解決記録 | `問い合わせフォーム送信失敗.md` |
| `changelog/` | リリース・運用上の更新履歴 | `2026-07.md` |
| `architecture/` | システム構成・設計判断 | `website-overview.md` |
| `prompts/` | 再利用するプロンプト | `seo-review.md` |
| `knowledge/` | 学習内容・技術メモ | `git-github.md` |

## 共通ルール

- 確認できた事実と、提案・推測を区別する
- APIキー、認証トークン、秘密鍵、個人情報を記録しない
- 同じ内容を複数の場所へ重複して保存しない
- ファイル名は内容が分かる短い日本語または英数字を使用する
- 重要な設計判断は、理由と影響範囲も残す
