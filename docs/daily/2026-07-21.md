# S3DOT 開発日報

## 日付

2026年7月21日

## 本日の目的

- AI導入支援LPを営業用途に合わせて再構築し、スマートフォン表示を仕上げる
- 既存サイトからAI業務改善LPへの導線を追加する
- Safari固有のContact見出し表示を修正する
- Google Tag Managerを全ページへ導入する

## 実施内容

### AI導入支援LPの再構築

- 重複していたBenefitsと中間CTAのセクションを削除・整理
- 課題、対象企業、対応業務、サービス内容、導入プロセス、費用、FAQ、診断、CTAを中心とする営業LPへ全面再構築
- LINE相談ボタン、FAQ構造化データ、信頼補足、診断セクションなどを追加
- セクションの新設・削除・並び替えと文章の書き直しを実施
- セッション上限で中断した作業を再開し、実装済み部分を変えず未完了の確認から継続

### スマートフォン表示

- 固定CTAがFooterやコンテンツと重なる問題を修正
- Hero、対象企業、診断、料金などの情報量・余白・見出しサイズを調整
- スマートフォンで消えていたネットワーク背景装飾をPCと同等の濃さへ戻した

### サイト内導線

- トップページのServiceセクションとFooterから `/ai` へ移動できるリンクを追加
- サービスページでも共通Serviceセクション経由でAI業務改善LPへ案内できる構成にした

### Safari表示

- Safari / WebKitでContactセクションの見出しが巨大化し、内容と重なる不具合へ再描画対策を追加
- 共通レイアウトからSafari向けの再描画処理を読み込む構成にした
- Chromiumでは再現せず、Safari実機でウィンドウ幅を変えると直るというユーザー共有情報をもとに、WebKitの初回レイアウト計算を原因候補として切り分け

### 計測

- Google Tag Managerを共通レイアウトへ追加し、全ページで読み込む構成にした

## 問題と解決

- 問題: LP内に重複する説明があり、営業導線として情報量と順序に改善余地があった
- 解決: セクションを整理し、対象企業、業務、費用、FAQ、診断まで含む構成へ再設計
- 問題: スマートフォン固定CTAがコンテンツやFooterに重なった
- 解決: 固定位置、余白、表示条件を調整
- 問題: スマートフォンで背景装飾が消えていた
- 解決: モバイル時の非表示・濃度指定を修正
- 問題: SafariでContact見出しが巨大化し、ほかの内容と重なった
- 解決: WebKit向けの再描画処理を追加
- 問題: セッション上限によりLP調整作業が中断した
- 解決: 再開時にGit状態と実装済み項目を先に確認し、未完了の検証だけを継続
