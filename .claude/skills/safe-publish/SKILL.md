# Safe Publish

## 目的
コミット・push前の安全確認を徹底し、壊れたコードや意図しない変更を本番に持ち込まない。

## 手順(順番厳守)
1. `git status` で変更・未追跡ファイルを確認する
2. `git diff` / `git diff --staged` で差分内容を確認する(check-changes Skillの観点も参照)
3. `npm run lint` を実行し、エラー・警告がないか確認する
4. `npm run build` を実行し、ビルドが成功するか確認する
5. 上記いずれかで問題が見つかった場合は、コミットせずに問題点を日本語で報告する
6. すべて問題なければ、以下を日本語で提示する
   - ステージすべきファイル一覧
   - 簡潔な日本語のコミットメッセージ案
7. 変更ファイルだけをステージし、日本語のコミットメッセージでコミットする
8. 現在のリモートブランチへpushする
9. `git status` でローカルとリモートが同期し、作業ツリーがクリーンであることを確認する
10. 変更内容、lint/build、コミット、pushの結果を日本語で報告する

## 禁止事項
- ユーザーがコミットまたはpushを行わないよう指定した場合は実行しない
- `.env*` や、APIキー・秘密鍵らしき値を含むファイルをステージしない(含まれている場合は警告して除外を提案する)
- lintまたはbuildが失敗した状態でコミットしない
- ユーザーの承認なしに `--no-verify` 等でチェックをスキップしない
- 仕様・デザイン・公開範囲などの判断が必要な場合は、コミット・pushせずに停止して確認する

## 完了後
- `git status` を実行し、ローカルとリモートが同期していることを確認する
