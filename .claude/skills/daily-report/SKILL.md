# Daily Report — 開発日報

## 目的

当日の作業、確認結果、問題と解決方法、判断事項、次の予定を `docs/daily/YYYY-MM-DD.md` に残す。

## 手順

1. 現在の日付とタイムゾーンを確認する
2. 対象日の既存日報があれば、最初に全文を読み、手書きの所感やメモを保持する
3. `git status` と対象日の `git log --stat` を確認する
4. Gitで確認できた作業と、ユーザーから共有されたGit外の作業を区別して整理する
5. `docs/daily/_TEMPLATE.md` の構成に沿って日報を作成または更新する
6. 推測で作業内容や成果を追加しない。不明な点は「未確認」と記載する
7. APIキー、認証トークン、秘密鍵、メールアドレスなどの秘密情報・個人情報を記載しない
8. 同じ日の日報ファイルを重複して作成しない

## 記載基準

- **本日の目的**: その日に達成しようとしたこと
- **実施内容**: 実際に行った作業。ツールや領域ごとに整理する
- **変更・コミット**: Gitで確認できた変更とコミット
- **確認結果**: lint、build、ブラウザ確認など
- **問題と解決**: 発生した問題、原因、対処
- **判断・学び**: 今後も参照する価値がある決定事項
- **次に行うこと**: 未完了事項と次回の優先順位
- **所感**: ユーザーが共有した内容のみ。既存内容は削除しない

## 完了報告

更新した日報のパスと、追加・更新した主な項目を日本語で報告する。
